Always wanting to have a website of my own for self-fulfilling reasons, I decided I will create one. By "create", I mean I will be the one to code (almost) everything...from scratch. Below are the site's features:

Created with separated Authed and Un-authed views

Perhaps the feature I really worked myself off for was this one. I literally spent days separating what a logged-in user versus an anonymous user can see and do...previously.

(Note: this will feel liek a blog post. :D )
Based on the latest trends in marketing, every successful (online) business today started showing its target market their product's or service's "a taste of what it's like". Proof? Free 7-day or 30-day trials.

Please log-out of facebook or any other social media account you have and try to search for any account you know of.
What did you notice? The account's preview still shows up even if you're not logged-in, right?
Maybe try searching for an item from an online shopping website on an incognito tab?

Prior to this concept, you'll need to "be a part of the community" first to view a site's pages. This tells you how old I am, I guess?
You can also try browsing any item on any online selling platform you know of and you'll be asked to log in or create and account only when you're actually ordering something.

The marketing game's now changed...
And I adapted it here. Custom-coded.

Social Media-like

Since I originally intended this site to be a forum, user accounts created are intended to have to display "distinct user info" (I will not call it "personal info" since they can all be dummy information just for users to have something to display). It'll be up to the users to add personal information should they determine that it will help people reach-out to them when needed.

Cross-Site Request Forgery (CSRF) protection

Another reason I stick to the Django framework was because I was convinced of its built-in security features.

Since this site features allowing users other than the admin to be writers/bloggers, as well, we need a better way of protecting unsuspecting victims of online frauds using HTML/SQL injections. I am convinced that Django's security features are what we need to minimize the risks. Going a little technical mentioning it here, you can check this out for more information. The first paragraph will give you a summary of what you need to know.

Google Login

As for most websites today, users can easily create their accounts using third-party providers' account APIs. Here, I tested-out Google and it's working fine. *sigh*
