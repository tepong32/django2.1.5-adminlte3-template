About teppy.rocks

Here's the part specifically intended for curious individuals. This contains a summary of the features / descriptions of everything that can be found/noticed here, as well as those I intend to add in the near future. (I only wanted to have this online ASAP. Functionalities are not yet implemented but the site, as a whole, is functional.)

Always wanting to have a website of my own for self-fulfilling reasons, I decided I will create one. By "create", I mean I will be the one to code (almost) everything...from scratch. This is the result of the countless tutorials I watched, trial versions of the project I made, Googling how to solve error results when trying to run the project, countless modifications to specific features I wished to implement, and everything else. (If you wish to check out the very simple, oldest versions of this website deployed, click this or this. See Log-in and Register pages. Everything stepped-up from there.)

Below are the site's features:

Created with separated Authed and Un-authed views

Perhaps the feature I really worked myself off for was this one. I literally spent days separating what a logged-in user versus an anonymous user can see (and do).

(Note + bragging rights: I was a business course student.)
Based on the latest trends in marketing, every successful (online) business today started showing its target market their product's or service's "a taste of what it's like".

Proof?

Please log-out of facebook or any other social media account you have and try to search for any account you know.
What did you notice? Your account's preview still shows up even if you're not logged-in, right?
Prior to this style of marketing, you'll need to "be a part of the community" first to view a member's profile.
You can also try browsing any item on any online selling platform you know of and you'll be asked to log in or create and account only when you're actually ordering something.

The marketing game's now changed...
And I adapted it here. Custom-coded.

Social Media-like

Since I originally intended this site to be a forum, user accounts created are intended to have to display "distinct user info" (I will not call it "personal info" since they can all be dummy information just for users to have something to display). It'll be up to the users to add personal information should they determine that it will help people reach-out to them when needed.

Cross-Site Request Forgery (CSRF) protection

Another reason I stick to the Django framework was because of its built-in security features.

(Disclaimer: No ill intent included.)
Unlike the usual language/framework that can be used to create static websites, since this site features allowing users other than the admin to be writers/bloggers, as well, we need a better way of protecting unsuspecting victims of online frauds using html/SQL injections. I am convinced that Django's security features are what we need to minimize the risks. Going a little technical mentioning it here, you can check this out for more information. The first paragraph will give you a summary of what you need to know.

Google Login

As for most websites today, users can easily create their accounts using third-party providers' account APIs. Here, I tested-out Google and it's working fine. *sigh*
